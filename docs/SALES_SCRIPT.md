# Sales Script

Use this script when presenting the E-commerce Starter Pack to a potential client.

## 30-Second Pitch

This is a ready-made e-commerce website system for small businesses in Bangladesh. Customers can browse products, add to cart, place orders with COD, bKash, or Nagad, and track their order. You get a private admin panel to manage products, upload images, update stock, verify manual payments, and manage orders.

## 2-Minute Pitch

Many small businesses depend only on Facebook or Instagram, so customers have to message for prices, availability, and order updates. This starter pack gives your business a professional website with a real product catalog, cart, checkout, and admin dashboard.

You can manage products yourself, upload photos, update prices, control stock, and see orders in one place. Customers can order with COD or manual bKash/Nagad, then track the order using order number and phone. The system is built on Supabase and Next.js, so it is a serious foundation that can grow later with blog, customer accounts, SMS, or real payment gateways.

## Client Discovery Questions

- What products do you sell?
- How many products do you want to launch with?
- Do you sell mostly through Facebook, Instagram, WhatsApp, or a physical shop?
- Do you need Bangla, English, or both?
- What delivery zones and delivery fees do you use?
- Do you accept COD?
- Do you use bKash, Nagad, or both?
- Are those accounts personal or merchant accounts?
- Who will manage products and orders?
- Do you already own a domain?
- Do you want email notifications at launch?
- Do you need future features like SMS, blog, customer accounts, or payment gateway?

## bKash/Nagad Manual Payment Explanation

This version does not connect directly to bKash or Nagad gateway. Customers place an order and choose bKash or Nagad. The owner checks the payment manually and marks payment as verified in the admin panel. Real gateway integration can be added later if the business needs automatic payment confirmation.

## Order Tracking Explanation

Customers do not need an account. They enter their order number and phone number on the tracking page. This protects private data while still letting customers see order and payment status.

## Admin Panel Explanation

The admin panel is private. Admin users are created manually and must be allowlisted. From admin, the owner can manage products, upload images, update stock, manage variants, view orders, update order status, and verify manual payments.

## Optional Email Setup Explanation

Email notification hooks are implemented. Live delivery requires the client's domain to be verified in Resend. If the client does not have domain email ready yet, the website and order flow still work without email.

## Common Objections

**Why not just use a Facebook page?**  
Facebook is useful for marketing, but a website gives customers a clear catalog, cart, checkout, order tracking, and a professional brand presence. It also reduces repeated messages about price, stock, and order status.

**Can I manage products myself?**  
Yes. You can add, edit, archive, upload images, set price, set stock, and manage variants from the admin panel.

**Will I get orders directly?**  
Yes. Orders are saved in the admin panel with customer details, items, payment method, totals, and status.

**Do I need technical knowledge?**  
No for daily use. After setup, you use the admin panel. Technical setup like Supabase, domain, deployment, and email configuration is handled during launch.

**Can it support my business type?**  
Yes, if you sell catalog products. It works for baby products, fashion, cosmetics, seasonal products, simple food/product sellers, and small retail.
