# Screenshot Checklist

Use this checklist for portfolio, sales pages, social posts, and client proposals.

## Public Storefront

- Home page desktop
- Home page mobile
- Shop page with product grid
- Shop page with search/filter visible
- Product detail page
- Product image gallery
- Variant selection area
- Stock/inventory display
- Cart page
- Checkout page
- Order success state with order number
- Track order page before lookup
- Track order result
- Contact page

## Admin Panel

- Admin login page
- Admin dashboard
- Admin orders list
- Admin order detail
- Order status controls
- Payment verification controls
- Admin products list
- Product create/edit form
- Image upload section
- Inventory section
- Variant section
- Product archive/publish control

## Mobile Screenshots

- Mobile home
- Mobile shop
- Mobile product detail
- Mobile cart
- Mobile checkout
- Mobile track order
- Mobile admin order detail

## Screenshot Safety

- Do not show real admin credentials.
- Do not show real customer phone numbers or addresses.
- Do not show Supabase keys, Resend keys, or `.env.local`.
- Use demo orders with clearly fake customer information.
- Blur or crop browser bookmarks if they reveal private information.
- Use a clean browser profile if recording public sales material.
